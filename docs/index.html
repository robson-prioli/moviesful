Documentação da API

Versão 1: Listar Filmes

Endpoint: /api/v1/movies

Descrição: Retorna uma lista de filmes.

Método: GET

Parâmetros:

  • filter (opcional) - Filtra os filmes pelo título.
  • sort (opcional) - Ordena os filmes (asc/desc).
  • page (opcional) - Número da página.
  • limit (opcional) - Número máximo de resultados por página.

Autorização: Token Bearer é obrigatório.

Exemplo de Requisição:

GET /api/v1/movies?filter=Action&sort=desc&page=1&limit=10
Authorization: Bearer YOUR_BEARER_TOKEN

Versão 2: Listar Filmes com XML

Endpoint: /api/v2/movies

Descrição: Retorna uma lista de filmes em XML.

Método: GET

Parâmetros:

  • filter (opcional) - Filtra os filmes pelo título.
  • sort (opcional) - Ordena os filmes (asc/desc).
  • page (opcional) - Número da página.
  • limit (opcional) - Número máximo de resultados por página.
  • output (opcional) - Define o formato de saída (json/xml).

Autorização: Token Bearer é obrigatório.

Exemplo de Requisição:

GET /api/v2/movies?filter=Action&sort=desc&page=1&limit=10&output=xml
Authorization: Bearer YOUR_BEARER_TOKEN

Versão 3: Listar Filmes com JWT

Endpoint: /api/v3/movies

Descrição: Retorna uma lista de filmes após validar o JWT.

Método: GET

Parâmetros:

  • filter (opcional) - Filtra os filmes pelo título.
  • sort (opcional) - Ordena os filmes (asc/desc).
  • page (opcional) - Número da página.
  • limit (opcional) - Número máximo de resultados por página.
  • Authorization (obrigatório) - Token JWT no cabeçalho de autorização.

Exemplo de Requisição:

GET /api/v3/movies?filter=Action&sort=desc&page=1&limit=10
                            Authorization: Bearer YOUR_JWT_TOKEN
